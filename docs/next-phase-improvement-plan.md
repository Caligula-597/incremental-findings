> 目标：在现有可运行平台基础上，完成“可上线 + 可运营 + 可持续迭代”的三阶段升级。

- 产品负责人：确认流程规则、优先级、对外口径。

- 运营/编辑负责人：审稿规则、伦理策略、通知模板审校。

---

## 5. 风险清单与应对

1. **风险：配置缺失导致线上行为不一致**  
   应对：发布门禁必须校验关键 ENV。

2. **风险：RLS 与应用逻辑不一致导致写入失败**  
   应对：迁移脚本与策略 SQL 纳入 CI 校验。

3. **风险：功能很多但无回归测试导致发布不稳**  
   应对：先覆盖高价值链路的最小自动化测试。

4. **风险：编辑登录策略单一造成安全隐患**  
   应对：引入 token 轮换与失败告警机制。
