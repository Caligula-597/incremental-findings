# 内容接入与版权合规策略（预印本/期刊平台）

> 非法律意见。用于产品与工程执行的风险控制基线。

## 结论（给产品决策）

- **直接“扒全文铺首页”风险高**，很可能侵权（尤其未获授权的全文复制、图表复制、批量镜像）。
- **只接入 metadata（标题、作者、DOI、链接、摘要在许可允许范围内）+ 跳转原站**，风险明显可控。
- 每个平台都要看 API ToS / License，不能一刀切。

---

## 可做 / 不可做（红线）

### ✅ 推荐做法（低风险）
1. 仅拉取并展示：标题、作者、发布时间、DOI、来源链接、许可证信息。
2. 首页展示“卡片+跳转原文”，不托管全文 PDF。
3. 存储来源字段：`source_platform`、`source_url`、`license`、`retrieved_at`。
4. 给每条数据保留删除/下架机制（平台投诉后可快速处理）。

### ❌ 高风险做法（避免）
1. 未授权抓取并重发全文 PDF/HTML。
2. 复制图表、补充材料、封面图到你站内长期托管。
3. 绕过 robots 或登录墙抓取。
4. 不保留来源与许可证信息的二次分发。

---

## 平台接入分级建议

### A 级（优先）
- 元数据开放明确、API 文档清晰的平台（例如 Crossref metadata 体系）。
- 只做 metadata 聚合与检索。

### B 级（谨慎）
- 可做摘要/全文片段展示，但需要逐条检查 license 或平台条款。

### C 级（暂缓）
- 条款不清晰、禁止再分发、或对抓取限制严格的平台。

---

## 首页“聚合模式”推荐

### 模式 1：Metadata-only（建议默认）
- 首页仅展示卡片：标题、作者、日期、来源、DOI、关键词。
- 点击后跳转原站详情页。

### 模式 2：摘要增强（可选）
- 仅在 license 允许时展示摘要。
- 摘要旁强制展示来源与许可说明。

### 模式 3：全文镜像（默认禁用）
- 仅在签署授权协议后开启。

---

## 工程落地清单

1. 建立 `source_registry`（平台白名单）
   - 字段：`platform`, `api_base`, `tos_url`, `license_policy`, `allowed_fields`, `risk_level`。
2. 建立 `ingestion_logs`
   - 记录每次抓取来源、字段、许可证判断、是否命中红线。
3. 建立 `takedown_workflow`
   - 接受投诉 → 24 小时内下架 → 审核复原。
4. 页面展示合规信息
   - 每条内容显示 `Source` + `License` + `Original Link`。

---

## 建议你现在的执行策略

- **短期（立即）**：只做 metadata 聚合，不做全文托管。
- **中期**：按平台逐个评估摘要展示权限。
- **长期**：与重点平台签约后再做深度聚合/镜像。
